## 📌 Описание проекта
В данной лабораторной работе реализована объектно-ориентированная модель базовых сущностей для стримингового музыкального сервиса. Архитектура приложения включает в себя два основных независимых класса, описывающих цифровой контент и пользователей платформы.

---

## ⚙️ Структура классов и вариантов

### 1. Класс `AudioContent` (Основная сущность)
Представляет собой цифровой аудиоматериал (музыкальный трек, подкаст или альбом), доступный на платформе.

*   **Атрибуты объекта:**
    *   `id` (`int`) — Уникальный идентификатор контента в базе данных.
    *   `title` (`str`) — Название аудиозаписи.
    *   `artist` (`str`) — Имя исполнителя, группы или автора подкаста.
    *   `duration` (`int`) — Длительность аудиозаписи в секундах.
*   **Методы:**
    *   `display_info()` — Консольный вывод всех характеристик объекта в структурированном виде.
    *   `__str__()` — Магический метод для красивого форматирования данных (переводит секунды в формат `Минуты:Секунды`).

### 2. Класс `Listener` (Субъект действия)
Описывает зарегистрированного пользователя платформы, который потребляет контент.

*   **Атрибуты объекта:**
    *   `id` (`int`) — Уникальный идентификатор пользователя.
    *   `username` (`str`) — Никнейм / имя аккаунта.
    *   `subscription_type` (`str`) — Текущий тарифный план пользователя (`Free` / `Premium`).
*   **Методы:**
    *   `__str__()` — Магический метод для компактного текстового представления профиля слушателя в логах или списках.
